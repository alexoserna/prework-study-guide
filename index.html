Prework Study Guide

HTML

  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.
  • Semantic HTML indicates the role of the content and gives structure to the page.
  • Div and Span would not be considered semantic HTML, but are useful for inline properties/content.
  • Within the head element you would put the title and link elements to give the page a title and link the CSS stylesheet.
  • Within the HTML tag you can adjust and set the default language for the page.
  • Header elements (h1-h4) should be used in order and there should only be one h1 element at the beginning of the page.

CSS

  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • Targetting multiple elements is possible by putting a comma after the first element then continuing to the second.
  • Color, size, shape, font style, and background color can all be changed through the use of CSS.
  • CSS can also be edited in the HTML instead of in a separate stylesheet.
  • Classes and Id's are useful when selecting/targetting an element to edit.

Git

  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it
  • git ls: tells you every file that exists within the current directory
  • git cd folder-name: will open the directory to the directed folder
  • git mkdir direcotry-name: will create a new directory following the name it is given
